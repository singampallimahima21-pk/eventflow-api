# Project Overview

## Purpose

`eventflow-api` is a lightweight backend service for ingesting user event data, validating and normalizing it, and storing it in a relational datastore. The API exposes endpoints for single-record processing, batch processing, and secure retrieval of persisted user records.

## Architecture

The application follows a layered design:

- `routes` — defines HTTP endpoints and request handling
- `schemas` — validates incoming request data with Pydantic models
- `services` — contains the core business logic and processing rules
- `models` — declares the database schema with SQLAlchemy ORM
- `db` — manages database engine and session lifecycle
- `core` — provides shared utilities, currently JWT authentication

## Request flow

1. A client submits data to a route in `app/routes/user_routes.py`.
2. FastAPI validates the request against the Pydantic schema in `app/schemas/user_schema.py`.
3. The route forwards normalized input to `app/services/user_service.py`.
4. The service validates business rules, normalizes data, and persists valid users via SQLAlchemy.
5. Responses include structured success or failure details.

## Key behaviors

- Data normalization: names are title-cased and email text is normalized.
- Validation rules: required name/email/age values, valid email format, age range 0–120.
- Persistence: user records are saved to SQLite using `app/models/user_model.py`.
- Security: `/api/users` requires a bearer JWT token generated by `/api/login`.

## Current limitations

- Login is implemented as a dummy endpoint and does not authenticate real users.
- Database is local SQLite; production should replace it with PostgreSQL or another robust datastore.
- The token secret is hardcoded and should be migrated to environment configuration.

## Extension opportunities

- Add real user authentication with hashed credentials.
- Replace SQLite with PostgreSQL or MySQL.
- Move configuration values into environment variables.
- Add OpenAPI examples and better error handling.
- Implement unit tests for service and route logic.

## Run instructions

Start the service with:

```powershell
uvicorn app.main:app --reload
```

Then use the OpenAPI docs at `http://127.0.0.1:8000/docs` to explore available endpoints.
